# TestLens Local Dashboard (4.0+)

The TestLens dashboard is a single-file, zero-dependency static SPA
that renders TestLens artifacts (`agent.json`, `advice.json`) in the
browser. It is served by `mix lens.serve` using Erlang's built-in
`:inets` HTTP server.

## Enabling

```sh
mix lens.serve                       # port 4000, serves _build/test_lens
mix lens.serve --port 4100           # override port
mix lens.serve --dir tmp/test_lens   # override artifact directory
```

Then open <http://localhost:4000>.

## Tabs

The dashboard has five tabs:

| Tab | Source | What it shows |
|---|---|---|
| Summary | `agent.json` | Totals, TestLens version, run info, counts. |
| Failures | `agent.json` | Per-failure details: file, classification, hypotheses, rerun. |
| Repair Queue | `agent.json` | Ranked repair targets with first-checks and verification. |
| OTP Snapshots | `agent.json` | Per-failure runtime context (when `--snapshot` was used). |
| Architecture Findings | `agent.json` ∪ `advice.json` | Static AST + topology findings (when `--advise` was used). |

## Files

The dashboard generates three files into the target directory:

| File | Purpose |
|---|---|
| `index.html` | Page skeleton, tab navigation, mount point. |
| `app.css` | Styling. Severity classes, responsive grid, dark-mode support. |
| `app.js` | Self-contained vanilla JS: fetches artifacts, renders tabs. |

All three are generated by `TestLens.Dashboard.generate/1`. The JS uses
no framework, no build step, no external assets.

## Privacy and stability

- The dashboard does NOT execute the JSON via `eval` or `innerHTML`
  except for trusted keys. Module names and rule IDs are inserted via
  `textContent` / `setAttribute` to prevent injection.
- Tab switching is purely client-side. No data leaves the browser.
- The `:inets` server binds to `127.0.0.1` only (default).

## Limits

- v4.0 ships `:inets` because it's stdlib. `:bandit` will replace it in
  v4.1 once stable.
- The dashboard is dev-only. `mix lens.serve` refuses to run outside
  `Mix.env() == :dev`.

## Stopping the server

Press `Ctrl-C`. The task traps SIGINT and stops `:inets` cleanly.
